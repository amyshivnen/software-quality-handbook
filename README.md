# software-quality-handbook
This repository contains a handbook of best practices for maintaining software quality in a fast-moving startup environment.

The handbook is intended for new and existing engineers to understand the standard development practices used within the company.

## Topics Covered

- Task Estimation in Scrum
- Code Reviews
- Automated Testing

## Repository Structure

| File | Description |
|-----|-------------|
| project-plan.md | Project planning and task allocation |
| task-estimation.md | Scrum estimation best practices |
| code-reviews.md | Code review guidelines |
| automated-testing.md | Testing best practices |
| retrospective.md | Project reflection |
| contributions.md | Team contribution summary |

## Development Process

This project follows a **Trunk-Based Development workflow**:

- Feature branches are created for each task
- Changes are merged using Pull Requests
- All team members review pull requests
