# Software Quality Assurance Handbook

This repository contains a handbook of best practices for maintaining software quality in a fast-moving startup environment.

The purpose of this handbook is to provide clear, practical guidelines for engineers to follow in order to improve consistency, reduce defects, and ensure high-quality software delivery.

It is designed to be easy to skim read, with key practices, bad practices to avoid, and supporting resources.

---

## Topics Covered

- Testing (Unit & Integration Testing)
- CI/CD Pipelines
- Bug Tracking & QA Processes

## Development Process

This project follows a **Trunk-Based Development workflow**:

- A feature branch is created for each task  
- Work is committed to that branch  
- A Pull Request (PR) is opened  
- At least **two team members review the PR**  
- Changes are approved and merged into `main`
